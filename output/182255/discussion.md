# [MLIR] TOSA PrintOpAvailability pass crashes with abort in llvm::raw_ostream::SetBufferAndMode

**Author:** Zustin
**URL:** https://github.com/llvm/llvm-project/issues/182255
**Status:** Closed
**Labels:** crash, mlir:tosa
**Closed Date:** 2026-02-25T16:26:50Z

## Body

I encountered a crash in MLIR. Based on the crash log, it appears to be related to multi-threaded processing, and I cannot reproduce it deterministically. The crash occurs when running: `mlir-opt input.mlir --test-tosa-op-availability` with LLVM/MLIR version 23.0.0git. See the input program below.

#### Input Program
```llvm
module {
  func.func @main() -> i32 {
    %c0 = arith.constant 0 : i32
    %c1 = arith.constant 1 : i32
    %sum = arith.addi %c0, %c1 : i32
    %sum_f = arith.sitofp %sum : i32 to f64
    %mem = memref.alloca() : memref<f64>
    memref.store %sum_f, %mem[] : memref<f64>
    %loaded = memref.load %mem[] : memref<f64>
    %res = arith.fptosi %loaded : f64 to i32
    return %res : i32
  }
  func.func private @print_matrix(memref<*xf64>, i64, i64, i64) -> ()
  func.func private @printf(memref<*xi8>) -> i32
}
```

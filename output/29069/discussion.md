## Extended Description 


clang-built swift compiler breaks since clang revision 274385. 
Might be related to bug #&#8203;28603.

When building swift using clang r274385, the built swift asserts, and fails to build the swift runtime. With clang r274379 (revisions 274380-4 seem unrelated), swift is built successfully.

This issue reproduced with current version of swift (swift-DEVELOPMENT-SNAPSHOT-2016-06-20-a-1538-ge657ece) but also with much older versions (Aug/15).

## Comments

### Comment 1 - llvmbot

Well, I have noticed and fixed several user code bugs that are revealed by the change r274385. Those bugs are mostly like:
```cpp
  struct B { A* a; };
  struct A { A(); };

  B Foo(const A& a) {
    return B(&a);  // Boom: keeps the pointer to a temporary.
  }

  int main() {
    B b = Foo(A());
    // b.a is dangling.
  }
```
